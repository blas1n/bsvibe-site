## 아키텍처

```
브라우저
   ↓
bsvibe.dev (Astro SSR on Vercel)
   ├─ /pricing                       ← Supabase에서 products + prices fetch
   ├─ /api/payment/checkout (POST)   ← Stripe/Toss 세션 생성
   ├─ /api/payment/webhook (POST)    ← 결제 이벤트 수신 → DB 업데이트
   └─ /api/payment/subscription      ← 구독 조회/취소
                                       ↓
                              Supabase Postgres
                              (products, prices, subscriptions, payment_history)
```

- **데이터**: Supabase Postgres에 모든 상품/가격/구독/결제내역 저장
- **결제**: 한국 → 토스페이먼츠, 해외 → Stripe (`getProvider(locale)`)
- **인증**: 기존 `bsvibe_session` httpOnly 쿠키 + `validateSession()` 미들웨어

## 1. Supabase 설정

### 1.1 마이그레이션 실행

`supabase/migrations/20260408_init_billing.sql` 파일이 준비되어 있습니다.

**Option A: Supabase Dashboard에서 수동 실행 (가장 간단)**
1. Supabase Dashboard → SQL Editor
2. 마이그레이션 파일 내용을 붙여넣기
3. Run

**Option B: Supabase CLI 사용**
```bash
npx supabase link --project-ref <your-project-ref>
npx supabase db push
```

마이그레이션은 다음 테이블을 생성합니다:
- `products` — BSVibe 4개 제품
- `prices` — 각 제품의 Free/Pro/Enterprise 티어
- `subscriptions` — 사용자 구독
- `payment_history` — 결제 이벤트 로그

기본 시드 데이터(제품 4개, 가격 12개)도 함께 삽입됩니다. Pro 가격은 `null`로 시작하므로, 가격 확정 후 SQL UPDATE로 수정하세요.

### 1.2 가격 수정 예시

```sql
update public.prices
set amount_krw = 9900, amount_usd_cents = 990
where product_id = 'bsgateway' and tier = 'pro';
```

### 1.3 환경 변수

`.env`에 추가:

```bash
SUPABASE_URL=https://<project-ref>.supabase.co
SUPABASE_ANON_KEY=eyJhbGc...           # public read 권한
SUPABASE_SERVICE_ROLE_KEY=eyJhbGc...   # 서버 전용 (구독/결제 쓰기)
```

Vercel에 배포할 때도 동일한 키를 환경 변수로 등록해야 합니다.

## 2. Stripe 설정

### 2.1 계정 및 키 발급

1. [Stripe Dashboard](https://dashboard.stripe.com) 가입
2. **Developers → API keys**에서 테스트 키 복사:
   - `sk_test_...` → `STRIPE_SECRET_KEY`
3. **Developers → Webhooks → Add endpoint**:
   - URL: `https://bsvibe.dev/api/payment/webhook`
   - Events:
     - `checkout.session.completed`
     - `invoice.paid`
     - `invoice.payment_failed`
     - `customer.subscription.updated`
     - `customer.subscription.deleted`
   - Signing secret 복사 → `STRIPE_WEBHOOK_SECRET`

### 2.2 Products & Prices 생성

Stripe Dashboard → Products에서 각 BSVibe 제품의 Pro 티어를 생성:
- BSGateway Pro (월간 구독)
- BSNexus Pro (월간 구독)
- BSupervisor Pro (월간 구독)
- BSage Pro (월간 구독)

각 Price ID(`price_xxx`)를 복사한 뒤 Supabase에서 매핑:

```sql
update public.prices
set stripe_price_id = 'price_1Abc...'
where product_id = 'bsgateway' and tier = 'pro';
```

### 2.3 환경 변수

```bash
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
```

### 2.4 로컬 웹훅 테스트

```bash
stripe listen --forward-to localhost:4321/api/payment/webhook
```

## 3. 토스페이먼츠 설정

### 3.1 계정 및 키 발급

1. [토스페이먼츠 개발자센터](https://developers.tosspayments.com) 가입
2. **테스트 키** 발급:
   - 시크릿 키: `test_sk_...` → `TOSS_SECRET_KEY`
   - 클라이언트 키: `test_ck_...` (프론트엔드용, 추후 PaymentWidget 페이지에서 사용)

### 3.2 웹훅 등록

토스 개발자센터 → 웹훅:
- URL: `https://bsvibe.dev/api/payment/webhook`
- 이벤트: `PAYMENT.DONE`, `PAYMENT_STATUS_CHANGED`
- 시크릿 → `TOSS_WEBHOOK_SECRET`

### 3.3 환경 변수

```bash
TOSS_SECRET_KEY=test_sk_...
TOSS_CLIENT_KEY=test_ck_...
TOSS_WEBHOOK_SECRET=...
```

### 3.4 PaymentWidget 페이지 (TODO)

토스는 Stripe와 달리 hosted checkout이 없습니다. `/account/billing/checkout` 페이지에서 PaymentWidget SDK를 로드하고, billing key 발급 후 `/api/payment/toss-confirm` 엔드포인트로 결제를 확정해야 합니다.

현재 코드는 구조만 잡아두고 있고, 실제 PaymentWidget UI는 키 발급 후 작성합니다.

## 4. 테스트

### 4.1 Stripe 테스트 카드
- 성공: `4242 4242 4242 4242`
- 실패: `4000 0000 0000 0002`
- 3D Secure: `4000 0025 0000 3155`

### 4.2 토스 테스트 카드
- 카드번호: `4330000000000001`
- 유효기간: 미래 아무 날짜
- CVC: 아무 3자리
