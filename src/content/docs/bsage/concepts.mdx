## 플러그인 vs 스킬

BSage는 두 가지 확장 메커니즘을 제공합니다:

| | 플러그인 | 스킬 |
|--|---------|------|
| 구현 | Python 코드 | Markdown (LLM 파이프라인) |
| 용도 | 외부 시스템 연동 | 텍스트 처리/분석 |
| 예시 | Telegram 봇, Email 수신 | 요약, 번역, 위험 분석 |

### 플러그인 카테고리

- **Input**: 외부에서 정보 수집 (Telegram, Slack, RSS 등)
- **Process**: 정보 가공 (요약, 분류, 태깅)
- **Output**: 결과 전달 (알림, 파일 저장, S3 업로드)

## Knowledge Graph

수집된 모든 지식은 그래프 구조로 연결됩니다:

- **노드**: 개별 지식 항목
- **엣지**: 지식 간 관계 (참조, 관련, 반박 등)
- **Vector Store**: 의미 기반 검색을 위한 임베딩 저장

## 2nd Brain 구조

BSage는 세 가지 영역으로 지식을 관리합니다:

1. **Seeds**: 새로 입력된 원시 정보
2. **Garden**: 정리되고 연결된 지식
3. **Actions**: 지식에서 도출된 실행 항목
