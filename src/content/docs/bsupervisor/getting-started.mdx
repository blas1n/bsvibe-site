---
title: 시작하기
description: BSupervisor에 가입하고 AI 에이전트 감사를 시작합니다.
---

BSupervisor는 AI 에이전트의 행동을 **실시간으로 감시하고 감사하는 안전 레이어**입니다. 에이전트가 수행하는 모든 작업을 이벤트로 수집하고, 사전 정의된 규칙에 따라 위협을 탐지하며, LLM 사용 비용을 추적합니다.

## BSupervisor가 하는 일

| 기능 | 설명 |
|------|------|
| **실시간 규칙 평가** | 8개 기본 안전 규칙으로 위협을 자동 감지 |
| **커스텀 규칙** | 조직의 보안 정책에 맞는 감사 규칙 생성 |
| **비용 추적** | 에이전트별/모델별 LLM API 비용 모니터링 및 이상 탐지 |
| **일일 보고서** | 규칙 위반, 비용 추이, 이상 탐지 결과를 자동 리포트 |
| **알림 연동** | Telegram, Slack으로 위반 사항 즉시 알림 |

## 빠른 시작

### 1단계: 계정 생성 및 로그인

1. 브라우저에서 [supervisor.bsvibe.dev](https://supervisor.bsvibe.dev)에 접속합니다.
2. **BSVibe 계정으로 로그인** 버튼을 클릭합니다.
   - 아직 BSVibe 계정이 없다면 [auth.bsvibe.dev](https://auth.bsvibe.dev)에서 회원가입합니다.
3. 로그인 후 대시보드가 표시됩니다.

![BSupervisor 대시보드 — 이벤트 수, 규칙 위반, 비용 추이를 한눈에 확인할 수 있습니다.](/images/screenshots/bsupervisor-dashboard.png)

대시보드에는 다음 정보가 표시됩니다:
- **Event Count**: 수집된 총 이벤트 수 (오늘/이번 주/이번 달)
- **Rule Violations**: 규칙 위반 건수와 심각도별 분포
- **Cost Overview**: 에이전트별 LLM 비용 차트
- **Recent Alerts**: 최근 발생한 알림 목록

### 2단계: 에이전트 연결

**Settings** 페이지에서 감시할 AI 에이전트 플랫폼을 연동합니다:

| 연동 방식 | 설정 방법 | 지원 플랫폼 |
|-----------|-----------|-------------|
| **BSVibe 자동 연동** | BSNexus 사용 시 자동으로 연결됨 | BSNexus |
| **API 키 기반** | 해당 플랫폼의 API 키를 입력 | OpenAI, Anthropic |
| **Webhook** | BSupervisor가 제공하는 Webhook URL을 에이전트에 설정 | 커스텀 에이전트 |

연동 절차:
1. **Settings > Integrations** 메뉴를 클릭합니다.
2. 연동할 플랫폼의 **Connect** 버튼을 클릭합니다.
3. 필요한 인증 정보(API 키 등)를 입력합니다.
4. **Test Connection** 버튼으로 연결을 확인합니다.
5. **Save** 버튼을 클릭하여 설정을 저장합니다.

### 3단계: 기본 규칙 확인

**Rules** 페이지로 이동하면 8개 기본 안전 규칙이 자동으로 활성화되어 있습니다:

![규칙 관리 페이지](/images/screenshots/bsupervisor-rules.png)

| # | 규칙 이름 | 심각도 | 설명 |
|---|----------|--------|------|
| 1 | **SQL Injection 탐지** | Critical | SQL 쿼리에 삽입 공격 패턴 감지 |
| 2 | **Prompt Injection 탐지** | Critical | 프롬프트 조작 시도 감지 |
| 3 | **PII 유출 방지** | Critical | 개인정보(이름, 이메일, 전화번호 등) 외부 전송 감지 |
| 4 | **과도한 리소스 사용** | Warning | CPU, 메모리, 네트워크 사용량 임계치 초과 감지 |
| 5 | **허가되지 않은 외부 API 호출** | Warning | 허용 목록에 없는 외부 API 호출 감지 |
| 6 | **민감 파일 접근** | Critical | `.env`, 인증서, 키 파일 등 민감 파일 접근 감지 |
| 7 | **비정상적 토큰 사용량** | Warning | 평균 대비 비정상적으로 높은 토큰 사용 감지 |
| 8 | **반복적 오류 패턴** | Info | 동일 에러가 반복 발생하는 패턴 감지 |

각 규칙은 개별적으로 활성화/비활성화할 수 있으며, 임계값과 알림 채널을 수정할 수 있습니다.

### 4단계: 알림 채널 설정

규칙 위반 시 즉시 알림을 받으려면 알림 채널을 설정합니다:

1. **Settings > Notifications** 메뉴를 클릭합니다.
2. 알림 채널을 추가합니다:
   - **Telegram**: Bot Token과 Chat ID를 입력합니다.
   - **Slack**: Slack Webhook URL을 입력합니다.
   - **Email**: 알림 수신 이메일 주소를 입력합니다.
3. 심각도별 알림 정책을 설정합니다:
   - **Critical**: 모든 채널로 즉시 알림
   - **Warning**: 선택한 채널로 알림
   - **Info**: 대시보드에만 기록 (알림 없음, 변경 가능)

### 5단계: 대시보드 모니터링

설정이 완료되면 **Dashboard**에서 다음 항목을 실시간으로 모니터링합니다:

- **이벤트 타임라인**: 시간순으로 정렬된 에이전트 이벤트 목록
- **규칙 위반 차트**: 일별/주별 위반 추이 그래프
- **비용 추이**: 에이전트별, 모델별 LLM 비용 차트

![BSupervisor 비용 추적 화면](/images/screenshots/bsupervisor-costs.png)
