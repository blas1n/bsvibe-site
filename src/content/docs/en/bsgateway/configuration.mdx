## Model Management

Add and manage LLM models on the **Models** page in the sidebar.

![Models Page](/images/screenshots/bsgateway-models.png)

### Adding a Model

Click the **Add Model** button and fill in the following fields:

| Field | Required | Description |
|-------|----------|-------------|
| Provider | Yes | Model provider — Anthropic, OpenAI, Google, Ollama, etc. |
| Model ID | Yes | Unique identifier used in API requests (e.g., `claude-sonnet`) |
| API Key | Yes | API key issued by the provider (stored with AES-256-GCM encryption) |
| Max Tokens | No | Maximum output tokens (uses provider default if not set) |
| API Base | No | Custom endpoint (for self-hosted models like Ollama) |

### Editing/Deleting Models

Use the **Edit** or **Delete** button next to each model in the list. When editing, you must re-enter the API key (the existing key is not displayed).

### Recommended Setup

We recommend registering at least 2 models:

| Role | Example Model | Use Case |
|------|---------------|----------|
| Low cost | gpt-4o-mini | Simple questions, translation, classification |
| High performance | claude-sonnet | Code generation, analysis, long-form writing |
| Premium | claude-opus | Complex reasoning, math, architecture design |

## Routing Configuration

Manage routing rules on the **Rules** page in the sidebar. See the [Routing Rules](/bsgateway/features/routing) page for detailed configuration.

## CORS Settings

If you're calling the BSGateway API directly from a frontend, you need to configure allowed domains.

In the dashboard, click ⚙️ → **Settings** in the top right:
1. Go to the **CORS Origins** section
2. Add your allowed domain (e.g., `https://myapp.com`)
3. Click **Save**

## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/chat/completions` | POST | OpenAI-compatible chat API |
| `/v1/models` | GET | List registered models |
| `/health` | GET | Service health check |
| `/health/ready` | GET | DB/Redis connection status |

### Chat API Request Format

```bash
POST https://api-gateway.bsvibe.dev/v1/chat/completions
Authorization: Bearer bsg_your_api_key
Content-Type: application/json

{
  "model": "auto",
  "messages": [
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"}
  ],
  "temperature": 0.7,
  "max_tokens": 1024
}
```

### Response Format

```json
{
  "id": "chatcmpl-...",
  "object": "chat.completion",
  "model": "claude-sonnet",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Hello! How can I help you today?"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 12,
    "total_tokens": 37
  }
}
```

> The `model` field in the response shows the model that was actually used. Even if you requested `auto`, the response returns the model that was selected.

## Error Codes

| Code | Description |
|------|-------------|
| 400 | Invalid request format |
| 401 | Invalid API key |
| 404 | Specified model is not registered |
| 429 | Rate limit exceeded |
| 500 | Internal server error |
| 502 | Provider API call failed |
