## Routing Pipeline

BSGateway processes every API request through a 4-stage pipeline. If a request matches at any stage, it is routed to that model; otherwise it falls through to the next stage.

```
Request received → ① Passthrough → ② Alias → ③ Pattern → ④ Auto → Model call
```

### ① Passthrough

If the `model` parameter in the request exactly matches a registered model ID, the request is forwarded directly to that model.

```json
{ "model": "claude-sonnet", "messages": [...] }
// → Forwarded directly to the registered claude-sonnet model
```

### ② Alias

Reference models through short aliases. Configure these on the **Rules** page in the dashboard.

| Alias | Target Model | Use Case |
|-------|-------------|----------|
| `fast` | gpt-4o-mini | Simple questions, quick responses |
| `smart` | claude-sonnet | Complex reasoning, code generation |
| `cheap` | local-llama | Minimum cost |

### ③ Pattern

Select a model based on request metadata or content using glob patterns. Configure patterns and priorities on the **Rules** page.

Examples:
- `code/*` pattern → claude-sonnet (code generation goes to a high-performance model)
- `translate/*` pattern → gpt-4o-mini (translation goes to a cheaper model)

Rules with higher priority are evaluated first.

### ④ Auto

Requests that don't match any of the above 3 stages are evaluated by the **Classifier**, which analyzes complexity and automatically selects the optimal model. Sending a request with `model="auto"` triggers this stage.

## Classifier

The classifier determines request complexity as one of `simple`, `moderate`, or `complex`.

| Complexity | Examples | Recommended Model |
|------------|----------|-------------------|
| Simple | "What's the weather?", simple translations | gpt-4o-mini (low cost) |
| Moderate | Code review, summarization, analysis | claude-sonnet (balanced) |
| Complex | Architecture design, math proofs, long-form generation | claude-opus (high performance) |

### Classification Strategies

Select a classification strategy in the **Rules** → **Auto Routing** section of the dashboard:

| Strategy | Behavior | Speed | Accuracy |
|----------|----------|-------|----------|
| **Static** | Keyword and pattern-based heuristics | 0ms | Moderate |
| **LLM** | An LLM evaluates complexity | ~200ms | High |
| **ML** | A trained classification model | ~10ms | High |

Static is the default. Use LLM or ML strategies when accuracy is critical.

## API Key Security

- **AES-256-GCM encryption**: Registered provider API keys cannot be viewed in plain text, even on the server
- **Multi-tenant**: Independent usage tracking and rate limiting per API key
- **Shown once at creation**: Keys cannot be retrieved after creation — issue a new one if lost

## Cost Tracking

Token usage and cost for every API request are automatically recorded:

- **Per-model cost aggregation**: See which models are used most and least
- **Time-based patterns**: Identify peak hours and usage trends
- **Routing efficiency analysis**: Estimate savings from auto routing

View data with period filters (Today / Last 7 days / Last 30 days) on the **Analytics** page.
