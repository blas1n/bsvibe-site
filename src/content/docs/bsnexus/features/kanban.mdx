## 개요

BSNexus의 Kanban 보드는 **WebSocket 기반 실시간 보드**로, Executor가 태스크를 처리하는 과정을 즉시 확인할 수 있습니다. 새로고침 없이도 태스크 카드가 칼럼 간에 자동으로 이동하며, 진행률과 로그가 실시간으로 갱신됩니다.

![BSNexus 대시보드의 Kanban 보드](/images/screenshots/bsnexus-dashboard.png)

## 칼럼 구성

Kanban 보드는 Task State Machine의 상태에 대응하는 4개 칼럼으로 구성됩니다:

| 칼럼 | 상태 | 설명 | 카드 색상 |
|------|------|------|-----------|
| **Backlog** | PENDING / ASSIGNED | 대기 중이거나 Executor가 배정된 태스크 | 회색 / 파란색 |
| **In Progress** | IN_PROGRESS | 현재 Executor가 작업 수행 중인 태스크 | 노란색 |
| **Review** | IN_REVIEW | 작업이 완료되어 검토를 기다리는 태스크 | 보라색 |
| **Done** | COMPLETED | 검토를 통과하고 최종 완료된 태스크 | 녹색 |

:::note
FAILED 상태의 태스크는 빨간색 배지와 함께 Backlog 칼럼에 표시됩니다. 자동 재시도 정책에 따라 다시 대기열에 진입하거나 수동 개입이 필요합니다.
:::

## 태스크 카드 정보

각 태스크 카드에는 다음 정보가 표시됩니다:

### 기본 정보
- **태스크 제목**: 작업의 간결한 이름 (예: "사용자 인증 API 구현")
- **태스크 설명**: Architect가 생성한 상세 요구사항
- **태스크 ID**: 고유 식별자 (예: `TASK-001`)

### 실행 정보
- **배정 Executor**: 현재 태스크를 처리 중인 Executor 이름
- **진행률**: 퍼센트 바로 표시되는 작업 진행도 (0~100%)
- **경과 시간**: 태스크가 IN_PROGRESS에 진입한 후 경과 시간

### 관계 정보
- **의존성 뱃지**: 이 태스크가 의존하는 다른 태스크의 수와 완료 상태
- **차단 뱃지**: 이 태스크의 완료를 기다리는 다른 태스크의 수
- **태그**: Architect가 부여한 분류 태그 (예: `frontend`, `api`, `database`)

### 상태 정보
- **재시도 횟수**: FAILED 후 재시도한 횟수 (해당되는 경우)
- **에러 메시지**: 최근 실패 원인 요약 (FAILED 상태인 경우)

## 카드 액션

태스크 카드를 클릭하면 상세 패널이 열리며, 다음 액션을 수행할 수 있습니다:

| 액션 | 설명 | 사용 가능 상태 |
|------|------|----------------|
| **상세 보기** | 태스크의 전체 요구사항, 실행 로그, 결과물 확인 | 모든 상태 |
| **로그 보기** | Executor의 실시간 작업 로그 스트리밍 | IN_PROGRESS |
| **결과물 보기** | 생성된 코드, 파일 목록, diff 확인 | IN_REVIEW, COMPLETED |
| **수동 재시도** | FAILED 태스크를 PENDING으로 되돌림 | FAILED |
| **태스크 수정** | 태스크의 제목, 설명, 요구사항 수정 | PENDING, FAILED |
| **의존성 수정** | 태스크 간 의존 관계 추가/제거 | PENDING |
| **태스크 삭제** | 태스크를 프로젝트에서 제거 | PENDING |

## 실시간 업데이트

### WebSocket 기반 자동 갱신

Kanban 보드는 WebSocket 연결을 통해 다음 이벤트를 실시간으로 반영합니다:

- **상태 전이**: 태스크 카드가 칼럼 간 자동 이동 (애니메이션 포함)
- **진행률 업데이트**: IN_PROGRESS 카드의 퍼센트 바가 실시간 갱신
- **로그 스트리밍**: 상세 패널에서 Executor의 작업 로그를 실시간 확인
- **새 태스크 추가**: Architect가 추가 태스크를 생성하면 즉시 보드에 반영
- **에러 알림**: 태스크 실패 시 토스트 알림과 함께 카드에 에러 배지 표시

### 동시 접속

같은 프로젝트를 여러 사용자가 동시에 열어도 모든 사용자에게 동일한 실시간 상태가 반영됩니다.

## 필터링 및 검색

보드 상단의 필터 바에서 다음 기준으로 태스크를 필터링할 수 있습니다:

| 필터 | 설명 | 예시 |
|------|------|------|
| **상태** | 특정 상태의 태스크만 표시 | IN_PROGRESS만 보기 |
| **Executor** | 특정 Executor에 배정된 태스크만 표시 | Claude Code Executor |
| **태그** | 특정 태그가 부여된 태스크만 표시 | `frontend` 태그 |
| **검색** | 제목/설명에서 키워드 검색 | "인증" 검색 |

## 수동 조작

필요한 경우 사용자가 직접 보드를 조작할 수 있습니다:

- **드래그앤드롭**: 태스크 카드를 다른 칼럼으로 드래그하여 상태를 수동 변경합니다.
- **우선순위 변경**: 같은 칼럼 내에서 카드 순서를 드래그로 변경합니다.
- **일괄 선택**: 여러 태스크를 선택하여 한번에 상태를 변경합니다.

:::caution
수동으로 태스크 상태를 변경하면 State Machine의 자동 전이와 충돌할 수 있습니다. IN_PROGRESS 상태의 태스크를 수동으로 COMPLETED로 변경하면 Executor의 작업이 중단됩니다.
:::
