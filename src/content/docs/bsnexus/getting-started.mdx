BSNexus는 AI 에이전트들이 협업하여 소프트웨어를 만드는 **오케스트레이션 플랫폼**입니다. 자연어로 아이디어를 설명하면 AI Architect가 프로젝트를 설계하고, 여러 Executor가 병렬로 작업을 수행합니다.

## BSNexus가 하는 일

| 기능 | 설명 |
|------|------|
| **AI Architect** | 자연어 대화를 통해 아이디어를 구조화된 프로젝트로 설계 |
| **자동 태스크 분해** | 프로젝트를 독립적인 작업 단위(Task)로 자동 분할 |
| **의존성 분석** | 태스크 간 의존 관계를 파악하여 실행 순서 결정 |
| **병렬 실행** | 여러 AI 에이전트(Executor)가 독립 태스크를 동시에 처리 |
| **실시간 Kanban** | WebSocket 기반 보드에서 전체 진행 상황을 실시간 추적 |

## 빠른 시작

### 1단계: 계정 생성 및 로그인

1. 브라우저에서 [nexus.bsvibe.dev](https://nexus.bsvibe.dev)에 접속합니다.
2. **BSVibe 계정으로 로그인** 버튼을 클릭합니다.
   - 아직 BSVibe 계정이 없다면 [auth.bsvibe.dev](https://auth.bsvibe.dev)에서 회원가입합니다.
3. 로그인 후 대시보드가 표시됩니다.

![BSNexus 대시보드 — 프로젝트 목록과 상태 요약이 한눈에 보입니다.](/images/screenshots/bsnexus-dashboard.png)

대시보드에는 다음 정보가 표시됩니다:
- **My Projects**: 생성한 프로젝트 목록
- **Recent Activity**: 최근 태스크 변경 이력
- **Active Executors**: 현재 작업 중인 에이전트 수
- **Quick Stats**: 완료율, 진행 중 태스크 수, 총 프로젝트 수

### 2단계: 프로젝트 만들기

1. 대시보드 우측 상단의 **+ New Project** 버튼을 클릭합니다.
2. 프로젝트 이름을 입력합니다 (예: `todo-app`, `blog-platform`).
3. 선택사항으로 설명을 추가합니다.
4. **Create** 버튼을 클릭하면 프로젝트가 생성되고 Architect 대화 화면으로 이동합니다.

### 3단계: Architect와 대화하기

Architect 페이지에서 만들고 싶은 것을 자연어로 설명합니다. 구체적일수록 더 정확한 태스크가 생성됩니다.

![Architect 대화 화면 — AI와 대화하며 프로젝트를 설계합니다.](/images/screenshots/bsnexus-architect.png)

**효과적인 프롬프트 예시:**

> "할 일 관리 웹앱을 만들어줘. React + TypeScript 프론트엔드, FastAPI 백엔드, PostgreSQL DB를 사용하고. 사용자 인증은 JWT 기반으로 해줘."

> "블로그 플랫폼을 만들어줘. 마크다운 에디터, 카테고리/태그 관리, SEO 최적화 기능이 필요해. Next.js로 만들어줘."

> "실시간 채팅 애플리케이션. WebSocket 기반, 채널 생성/참여, 메시지 검색 기능 포함. Python + Redis pub/sub 구조로."

Architect는 다음과 같은 과정을 거칩니다:

1. **요구사항 분석**: 설명을 파싱하여 핵심 기능을 추출합니다.
2. **명확화 질문**: 모호한 부분이 있으면 추가 질문을 합니다 (예: "인증 방식은 어떤 것을 원하시나요?").
3. **프로젝트 구조 생성**: 디렉토리 구조, 기술 스택, 모듈 구성을 설계합니다.
4. **태스크 분해**: 프로젝트를 독립적인 태스크로 분할하고 의존성을 설정합니다.
5. **Executor 배정**: 각 태스크에 적합한 Executor를 자동으로 배정합니다.

:::tip[프롬프트 팁]
- 사용할 기술 스택을 명시하면 더 정확한 결과를 얻습니다.
- 핵심 기능을 구체적으로 나열하세요.
- Architect가 질문하면 상세하게 답변할수록 결과가 좋습니다.
:::

### 4단계: 태스크 검토 및 실행

Architect가 태스크를 생성하면 **Board** 탭에서 Kanban 보드가 자동으로 구성됩니다:

1. 생성된 태스크 목록을 검토합니다.
2. 필요시 태스크를 수정하거나 추가/삭제합니다.
3. **Start Execution** 버튼을 클릭하면 Executor가 작업을 시작합니다.
4. Kanban 보드에서 각 태스크의 진행 상태를 실시간으로 확인합니다.

### 5단계: 결과 확인

모든 태스크가 완료되면:
- 각 태스크의 결과물(코드, 파일 등)을 확인합니다.
- 필요한 경우 Architect에게 추가 수정을 요청합니다.
- 프로젝트 결과를 다운로드하거나 연결된 저장소에 푸시합니다.
